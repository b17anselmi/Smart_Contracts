ERC827 Token Standard Walk-Through

The ERC827 Ethereum token standard is an extension of the ERC20 standard, meaning that it has all the functionality and attributes as the ERC20 standard, and some additional capabilities added on. One of the most noticeable additions is that with this standard, data in the form of bytes can also be sent to other addresses. A problem that the ERC20 standard has which the ERC827 standard addresses is that if tokens are sent to a smart contract which is not designed to handle that type of token, or tokens at all, that smart contract can’t access them, and you can’t get them back, so they’ll be lost.

Functions

  • function approveAndCall(address _spender, uint256 _value,bytes _data) public payable returns (bool);
    Allows _spender to withdraw from the account interacting with the smart contract (msg.sender) up to the amount indicated by _value. There are more checks set into this function, and it calls others from the ERC20 super class.
  • function transferAndCall(address _to, uint256 _value, bytes _data) public payable returns (bool);
    Sends _value amount of tokens from msg.sender to _to and fires a Transfer event which is logged in the blockchain.
  • function transferFromAndCall(address _from, address _to, uint256 _value, bytes _data) public payable returns (bool);
    Transfers tokens from the address specified by _from to the address specified by _to, firing a Transfer event. It is also payable, which means this function allows ether to be sent to the smart contract.
